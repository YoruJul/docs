## Why use an antinuke system?

Implementing an antinuke system is crucial to prevent moderators from misusing their permissions. This security feature helps ensure your server remains protected from harmful actions.

## How does the antinuke work?

The antinuke will set a limit on the number of actions a moderator can perform in a certain time frame. If the limit is exceeded, the moderator will be punished and a message will be sent to the owner.

## Configuring the antinuke

<Warning>
  Discord recently implemented a new **mass ban** feature which can quickly ban
  **100+ members**. It's **HIGHLY** recommended to utilizie [fake
  permissions](/security/fake-permissions) to prevent moderators from using this
  feature.
</Warning>

### Allowing users to configure the antinuke

Initially, only the **server owner** can configure the antinuke. However, you can allow other users to configure the antinuke with the `antinuke admin` command.

<Warning>
  This is a dangerous command and will allow the user to entirely alter the
  antinuke configuration.
</Warning>

<CodeGroup>
```javascript Syntax
,antinuke admin <add/remove> <@user>
```

```javascript Example
,antinuke admin add @xrm0s
```

</CodeGroup>

### Exempting users from the antinuke

You can exempt users from the antinuke with the `antinuke whitelist` command.

<Warning>
  This is a dangerous command and will allow the user to bypass the antinuke
  entirely.
</Warning>
<CodeGroup>
```javascript Syntax
,antinuke whitelist <add/remove> <@user>
```

```javascript Example
,antinuke whitelist add <@user>
```

</CodeGroup>

## Enabling an antinuke module

### Available Flags

The following flags can be used to define the antinuke module

    <Accordion title="Punishment">
        The punishment which will be applied to the moderator.
        <Info>Available punishments can be found in the [punishments](/resources/syntax#flag-punishments) section.</Info>
        <CodeGroup>
        ```javascript Syntax
        ,antinuke ban enable (punishment)
        ```

        ```javascript Example
        ,antinuke ban enable timeout
        ```
        </CodeGroup>
    </Accordion>
</AccordionGroup>

### Available Modules

<AccordionGroup>
    <Accordion title="Bot Protection">
        Control bot additions to your server
        
        <Info>Use `antinuke whitelist add/remove <userID>` to manage trusted users.</Info>
        <CodeGroup>
        ```javascript Syntax
        ,antinuke bot <enable/disable>
        ```
        ```javascript Example
        ,antinuke bot enable
        ```
        </CodeGroup>
    </Accordion>

    <Accordion title="Ban Protection">
        Prevent mass banning of members
        
        <CodeGroup>
        ```javascript Syntax
        ,antinuke ban <enable/disable>
        ```
        ```javascript Example
        ,antinuke ban enable
        ```
        </CodeGroup>
    </Accordion>

    <Accordion title="Channel Protection">
        Prevent unauthorized channel modifications
        
        <CodeGroup>
        ```javascript Syntax
        ,antinuke channel <enable/disable>
        ```
        ```javascript Example
        ,antinuke channel enable
        ```
        </CodeGroup>
    </Accordion>

    <Accordion title="Role Protection">
        Prevent role deletions and modifications
        
        <CodeGroup>
        ```javascript Syntax
        ,antinuke role <enable/disable>
        ```
        ```javascript Example
        ,antinuke role enable
        ```
        </CodeGroup>
    </Accordion>

    <Accordion title="Webhook Protection">
        Control webhook creation and modification
        
        <CodeGroup>
        ```javascript Syntax
        ,antinuke webhook <enable/disable>
        ```
        ```javascript Example
        ,antinuke webhook enable
        ```
        </CodeGroup>
    </Accordion>

    <Accordion title="Whitelist Management">
        Manage trusted users
        
        <Info>Whitelisted users bypass antinuke checks</Info>
        <CodeGroup>
        ```javascript Syntax
        ,antinuke whitelist <add/remove/list> [userID]
        ```
        ```javascript Example
        ,antinuke whitelist add 123456789
        ```
        </CodeGroup>
    </Accordion>

    <Accordion title="Reset Configuration">
        Reset all antinuke settings
        
        <CodeGroup>
        ```javascript Syntax
        ,antinuke reset
        ```
        ```javascript Example
        ,antinuke reset
        ```
        </CodeGroup>
    </Accordion>
</AccordionGroup>

## Disabling an antinuke module

You can disable an antinuke module with the same command you used to enable it, but with the status set to `disable`.

## Viewing the antinuke configuration

You can use the `antinuke config` command to view the current antinuke configuration.

### Viewing the modules & whitelist

You can use the `antinuke list` command to view the enabled modules and whitelisted users.

### Viewing users with antinuke admin

You can use the `antinuke admins` command to view users which can configure the antinuke.
